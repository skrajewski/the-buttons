Introduction

It's not an another pretty awesome CSS framework with many features and lots possibilities. It's not a set of magic CSS classes to solve every problem which you have with HTML and CSS at your work. It's not a ready-to-use CSS library. It's just an idea with a simple prototype to show how to design a maintainable, useful and good-looking component such as, in this case - button.

I'm not hiding that I was strong inspiring by Material Design by Google and other CSS Framework such as Bootstrap or Foundation. I focus on buttons only, because I would like to show how a considered start with a new project may bring many benefits in future.

Why buttons?

Besides content, buttons are very important because they encourage people to interact with the website or with the web application. Similar to a real world, when We see a button, we expect some action on pressing it. But there is one big difference between physical buttons and virtual buttons. First of them give a user a natural, physical feedback such as click sound, press or something else. We can touch this button, feel it, press strongly or slightly. Virtual buttons must be programmed to emit an artificial feedback to the user, and We, as web developers are responsible for doing it right.

Preface

Introduction

It's not an another pretty awesome CSS framework with many features and lots possibilities. It's not a set of magic CSS classes to solve every problem which you have with HTML and CSS at your work. It's not a ready-to-use CSS library. It's just an idea with a simple prototype to show how to design a maintainable, useful and good-looking component such as, in this case - button.

I'm not hiding that I was strongly inspiring by Material Design by Google and other CSS Framework such as Bootstrap or Foundation. I focus on buttons only, because I would like to show how a considered start with a new project may bring many benefits in the future.

Why buttons?

Besides content, buttons are very important because they encourage people to interact with the website or with the web application. Similar to a real world, when We see a button, we expect some action on pressing it. But there is one big difference between physical buttons and virtual buttons. First of them give a user a natural, physical feedback such as click sound, press or something else. We can touch this button, feel it, press strongly or slightly. Virtual buttons must be programmed to emit an artificial feedback to the user, and We, as web developers are responsible for doing it right. Also, things like labels, colors, and symbols are important to give buttons additional meaning. In short, We have many things to do when designing a seemingly simple item such as the button. Thus I decided to play with it!

Explore

The main purpose was redesigned an ugly, native browser button to fresh, modern and good-looking equivalent. It has custom behavior such as focus, hover, and active and should works fine with many other components and elements such as tables, cards, messages, etc. They also have a lot of additional classes which permit modify a final appearance.

I would like to encourage you to explore my experiments with buttons. My goal was to prototype highly accessible and meaningful buttons which could be used in many places on the website. I hope I meet some of these assumptions.
